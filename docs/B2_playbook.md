# B2 — מערכת החיסון: ספר נהלים (Playbook) לרישום ובטיחות

**מטרה:** פרוטוקולי תגובה, ניתוח וביקורת תחת מצבי SAFE/HOLD והפעלת רכיבי B2.

## תרחיש 1 — זיהוי פעולה מסוכנת (רכיב ל)
**זיהוי:** `ל` מזהה סיכון/אי-בטיחות רגעית בהחלטה מאושרת-מראש.  
**פעולה מיידית:**
- הפעלת **SAFE_on** → מעבר למצב **SAFE**.
- השעיית כל כתיבות מצב (No-Write-in-SAFE).
- חסימת הפעולה ו/או ניתובה ל-“רישום בלבד”.
**תגובת צוות:**
- קבלת התראה; ניתוח Ledger והסבר מדוע נחסם (טעם מדיניות).
- עדכון מדיניות דרך המסלול **Architect → A2 → A1** (לא עוקפים).

## תרחיש 2 — ביקורת/חקירה (רכיב כ)
**זיהוי:** צורך לשחזר/לנתח אירוע עבר.  
**פעולה מיידית:**
- שימוש ב-Audit (`כ`) לשליפת אירוע לפי מזהה/זמן/מטא-דאטה.
- אחזור רשומת המיזוג (3⊥+1) עם מקורות, תוצאה וחתימה.
**תגובת צוות:**
- שחזור נתיב הביקורת; **Rollback** מבוקר בסביבת בדיקות לפני/אחרי.
- הפקת דוח והסקת לקחים.

